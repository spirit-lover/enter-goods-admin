# Admin React UI/UX Redesign Design

## 背景

当前 React 管理端已经完成业务功能覆盖，但视觉表现仍偏默认 Ant Design 风格：

- 登录页结构完整，但品牌感和第一印象较弱
- 后台壳层具备基础布局，缺少企业控制台应有的秩序感与识别度
- Dashboard、列表页、系统页的内容容器和节奏较平，页面“完成了功能”，但没有形成统一产品感

本次优化仅调整前端 UI/UX 表达，不改接口契约、业务流程、权限模型与信息架构。

## 目标

为 `apps/admin-react` 建立一套“商务质感型”的企业后台视觉体系，使页面在不牺牲可读性、效率和可维护性的前提下，显著提升：

- 品牌识别
- 企业感与可信度
- 内容层级
- 数据页面的节奏和秩序
- 登录到后台内页的一致体验

## 非目标

- 不变更后端 API
- 不增加新业务模块
- 不修改路由语义
- 不引入新的 UI 框架
- 不重写为高度定制设计系统组件库

## 设计方向

### 风格定位

采用“稳健商务感”路线：

- 深海蓝作为品牌主轴
- 暖白和灰蓝作为主背景与内容色
- 低饱和铜金作为少量强调色
- 控制动效与装饰强度，避免科技炫光、过度玻璃感、泛滥渐变

最终效果应接近“企业 SaaS 控制台”而不是“展示型官网”。

### 设计原则

1. 数据优先
   页面要先服务于商品、库存、订单和系统治理信息，而不是被装饰喧宾夺主。

2. 层级清晰
   标题、摘要、工具条、指标、表格容器、操作按钮必须有明确等级，降低扫描成本。

3. 统一语义
   按钮、Tag、输入框、卡片、状态色、面板描边和阴影要说同一种视觉语言。

4. 克制表达
   只在登录页、品牌区、关键指标和少量状态点上使用更明显的视觉强调。

5. 保持 Ant Design 可维护性
   以主题覆盖、容器样式和页面结构优化为主，不做高风险深度魔改。

## 全局设计系统

### 色彩

- 主色：`#153A6B`
- 深色背景：`#0F2747`
- 页面底色：`#F5F1EA`
- 内容面板：`#FFFDF9`
- 主文字：`#142235`
- 次文字：`#5C6A7D`
- 边框：`#D7D3CA`
- 强调色：`#B68A52`

应用规则：

- 深海蓝用于侧栏、主按钮、重要标题强调
- 铜金只用于品牌高光、数字 accent、少量标签和选中边缘
- 危险和成功状态保留标准语义色，避免品牌色挤占功能反馈

### 背景与容器

- 全站背景为暖白基底叠加低强度径向光斑
- 内容区使用统一面板容器，具有轻描边、细阴影、圆角和弱高光
- 后台内页避免大面积玻璃模糊，仅保留顶部区域的轻微透明感

### 字体与层级

- 保持系统中文字体栈，确保兼容性和渲染稳定
- 标题更强调字重和间距
- 指标数字使用更强的字号与对齐策略
- 说明文案与辅助文本统一降噪，减少默认散乱感

### 交互

- 交互动效集中在 `160ms-220ms`
- 仅使用透明度、阴影和少量位移
- 输入框 focus、按钮 hover、菜单选中态保持统一品牌响应

## 页面级设计

### 登录页

登录页采用双栏门厅结构：

- 左侧为品牌叙事区：品牌标题、核心能力、摘要型运营指标或能力卡
- 右侧为登录卡：更强层次、更明确的输入焦点、更稳定的按钮与提示

目标是把登录页从“表单入口”提升为“后台产品入口”。

### 后台壳层

壳层保留侧栏 + 顶栏 + 内容区结构，但强化三个区域：

- 侧栏：品牌中轴，提升 logo 区、菜单容器和选中态质感
- 顶栏：收窄高度，增强信息密度和管理感
- 内容区：通过统一面板和页面头部结构建立秩序

### Dashboard

Dashboard 调整为运营驾驶舱风格：

- 顶部为高信息密度指标卡
- 下半区组织最近操作、库存预警、订单摘要等模块
- 模块间通过不同面板层级形成节奏，不再是简单卡片平铺

### 列表页与系统页

列表与治理页保持现有业务结构，但统一表达：

- 筛选区升级为工具条式面板
- 表格容器独立包裹，强化边界
- 行内操作更轻、更稳，不做颜色轰炸
- Tag、按钮、页头、抽屉保持同一商务语义

## 可访问性与响应式

- 保留足够的文本对比度
- 交互元素焦点态不可移除
- 平板和窄屏下允许头部与筛选区折行
- 不依赖 hover 才能理解操作

## 风险与控制

### 风险 1：视觉增强影响数据可读性

控制方式：

- 表格区域保持高对比和低装饰
- 控制装饰只出现在壳层、登录页和少量概览模块

### 风险 2：样式改动范围过大导致维护成本上升

控制方式：

- 优先集中在 `src/style.css` 的设计令牌和公共容器
- 页面结构仅做必要调整，不拆散业务逻辑

### 风险 3：Ant Design 默认主题与自定义样式冲突

控制方式：

- 通过外层 class 与页面容器覆盖为主
- 谨慎修改基础组件深层选择器

## 验收标准

- 登录页、壳层、Dashboard、列表页形成统一商务风格
- 页面层级比当前版本更清晰
- 不影响现有路由、权限和接口行为
- 仍通过现有 React 应用的测试、类型检查和构建验证
